## 🚀 1633. Percentage of Users Attended a Contest

### 💡 Idea

The goal is to calculate the percentage of all users who registered for each contest.

For each `contest_id`:

- Count the number of registered users.
- Divide it by the total number of users.
- Multiply by **100** to obtain the percentage.
- Round the result to **2 decimal places**.

Finally, sort the results by percentage in descending order and by `contest_id` in ascending order to break ties.

---

## 💻 Solution

```sql
SELECT contest_id,
    ROUND(
        COUNT(user_id) * 100 /
        (SELECT COUNT(*) FROM Users),
        2) AS percentage
FROM Register
GROUP BY contest_id
ORDER BY percentage DESC, contest_id;
```

---

### 🛠️ How It Works

1. Group the `Register` table by `contest_id`.
2. Count the number of registered users for each contest using `COUNT(user_id)`.
3. Use a subquery to calculate the total number of users.
4. Divide the number of registered users by the total users and multiply by **100**.
5. Round the percentage to **2 decimal places**.
6. Sort by:
   - Percentage (descending).
   - Contest ID (ascending).

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(single scan of `Register`; the scalar subquery for total users is evaluated once)*

**Space Complexity:** `O(n)` *(stores grouped contests)*
