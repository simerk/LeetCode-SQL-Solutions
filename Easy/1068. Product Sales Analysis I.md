## 🚀 1068. Product Sales Analysis I

### 💡 Idea

The goal is to report the **product name**, **year**, and **price** for each sale.

The `Sales` table contains the sales details, while the `Product` table stores the corresponding product names. We can join the two tables using the common `product_id` column.

---

### 💻 Solution

```sql
SELECT p.product_name, s.year, s.price
FROM Sales s
INNER JOIN Product p
ON s.product_id = p.product_id;
```

---

### 🛠️ How It Works

1. Join the `Sales` and `Product` tables using `product_id`.
2. Retrieve the `product_name` from the `Product` table.
3. Retrieve the `year` and `price` from the `Sales` table.
4. Return the required columns for each sale.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(single join between the two tables, assuming appropriate indexing)*

**Space Complexity:** `O(1)` *(excluding the output result set)*
