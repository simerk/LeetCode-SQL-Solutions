## 🚀 175. Combine Two Tables

### 💡 Idea

We need to display **every person** from the `Person` table, along with their address details if they exist.

A `LEFT JOIN` is the ideal choice because it returns **all records from the `Person` table**. If a person doesn't have a matching record in the `Address` table, the `city` and `state` columns are automatically returned as `NULL`, which is exactly what the problem requires.

---

### 💻 Solution

```sql
SELECT
    p.firstName,
    p.lastName,
    a.city,
    a.state
FROM Person p
LEFT JOIN Address a
ON p.personId = a.personId;
```

---

### 🛠️ How It Works

1. Start with the **Person** table, which contains all people.
2. Perform a `LEFT JOIN` with the **Address** table using `personId`.
3. For each person, SQL looks for a matching address.
4. If a match is found, the corresponding `city` and `state` are returned.
5. If no matching address exists, `city` and `state` are returned as `NULL`.
6. Finally, select the required columns: `firstName`, `lastName`, `city`, and `state`.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(depends on the join operation and indexing on `personId`)*

**Space Complexity:** `O(1)` *(excluding the output result set)*
