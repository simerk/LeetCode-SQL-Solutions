## 🚀 1934. Confirmation Rate

### 💡 Idea

The goal is to calculate the **confirmation rate** for each user.

The confirmation rate is defined as:

```text
(Number of 'confirmed' messages) / (Total confirmation requests)
```

If a user has **no confirmation requests**, their confirmation rate should be `0.00`.

We use a **LEFT JOIN** to include all users, even those without confirmation records, and compute the rate using conditional aggregation.

---

## 💻 Solution

```sql
SELECT s.user_id,
ROUND(
        IFNULL(AVG(c.action = 'confirmed'), 0),
        2 ) AS confirmation_rate
FROM Signups s
LEFT JOIN Confirmations c
ON s.user_id = c.user_id
GROUP BY s.user_id;
```

---

### 🛠️ How It Works

1. Start with the `Signups` table to include every user.
2. Perform a `LEFT JOIN` with the `Confirmations` table.
3. The expression:
   - `c.action = 'confirmed'`
   - evaluates to `1` for confirmed requests and `0` otherwise.
4. `AVG()` computes:
   - `(Number of confirmed requests) / (Total requests)`
5. `IFNULL(..., 0)` handles users with no confirmation records by returning `0`.
6. Round the result to **2 decimal places**.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)`

**Space Complexity:** `O(n)`
