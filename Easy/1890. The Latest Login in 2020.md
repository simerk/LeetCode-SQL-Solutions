## 🚀 1890. The Latest Login in 2020

### 💡 Idea

The goal is to find the **latest login time** for each user during the year **2020**.

We first filter the login records to include only those from **2020**, then group them by `user_id` and use `MAX()` to retrieve the most recent login timestamp.

---

## 💻 Solution

```sql
SELECT user_id, MAX(time_stamp) AS last_stamp
FROM Logins
WHERE YEAR(time_stamp) = 2020
GROUP BY user_id;
```

---

### 🛠️ How It Works

1. Filter the records to keep only logins from the year **2020**.
2. Group the remaining records by `user_id`.
3. Use `MAX(time_stamp)` to find the latest login time for each user.
4. Return the user ID along with the latest login timestamp.

---

## 💻 Approach 2: Using a Date Range

```sql
SELECT user_id, MAX(time_stamp) AS last_stamp
FROM Logins
WHERE time_stamp >= '2020-01-01' AND time_stamp < '2021-01-01'
GROUP BY user_id;
```

### 🛠️ How It Works

1. Filter records whose timestamps fall within the year 2020.
2. Group the records by `user_id`.
3. Use `MAX()` to retrieve the latest login timestamp for each user.

Using a date range avoids applying a function to the indexed column, making it more efficient for large datasets.

---

### ⏱️ Complexity

| Approach | Time | Space |
|----------|------|-------|
| `YEAR()` | `O(n)` | `O(n)` |
| Date Range | `O(n)` | `O(n)` |
