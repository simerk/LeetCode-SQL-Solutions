## 🚀 1581. Customer Who Visited but Did Not Make Any Transactions

### 💡 Idea

The goal is to find customers who visited the mall but **did not make any transactions**.

We perform a **LEFT JOIN** between the `Visits` and `Transactions` tables using `visit_id`. If a visit has no matching transaction, the transaction fields will be `NULL`. We then group the results by customer and count the number of such visits.

---

## 💻 Solution

```sql
SELECT v.customer_id, COUNT(*) AS count_no_trans
FROM Visits v
LEFT JOIN Transactions t ON v.visit_id = t.visit_id
WHERE t.transaction_id IS NULL
GROUP BY v.customer_id;
```

---

### 🛠️ How It Works

1. Start with the `Visits` table to include every visit.
2. `LEFT JOIN` the `Transactions` table using `visit_id`.
3. Visits without a matching transaction will have `NULL` in `transaction_id`.
4. Filter these visits using `WHERE t.transaction_id IS NULL`.
5. Group the remaining rows by `customer_id`.
6. Count the number of visits without transactions for each customer.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(single join followed by grouping)*

**Space Complexity:** `O(n)` *(stores grouped customers)*
