## 🚀 3465. Find Products with Valid Serial Numbers

### 💡 Idea

The goal is to find all products whose **description** contains a valid serial number.

A valid serial number must:

- Start with **`SN`** (case-sensitive).
- Be followed by **exactly 4 digits**.
- Contain a **hyphen (`-`)**.
- End with **exactly 4 digits**.
- Appear anywhere within the description.

We use the `REGEXP_LIKE()` function along with **word boundaries (`\b`)** to search for the required pattern.

---

## 💻 Solution

```sql
SELECT *
FROM Products
WHERE REGEXP_LIKE (description, '\\bSN[0-9]{4}-[0-9]{4}\\b', 'c')
ORDER BY product_id;
```

---

### 🛠️ How It Works

1. Use `REGEXP_LIKE()` to search the `description` for a valid serial number.

2. The regular expression used is:

   ```regex
   \bSN[0-9]{4}-[0-9]{4}\b
   ```

3. `\b`

   Represents a **word boundary**.

   It ensures the serial number starts at the beginning of a word and is not part of another word.

   **Example:**

   ```
   Product SN1234-5678     ✅
   ABCSN1234-5678          ❌
   ```

4. `SN`

   Matches the uppercase letters **`SN`** exactly.

5. `[0-9]{4}`

   - `[0-9]` → Matches any digit from `0` to `9`.
   - `{4}` → Means **exactly 4 digits**.

   Examples:

   ```
   1234
   0001
   9876
   ```

6. `-`

   Matches the literal hyphen between the two groups of digits.

7. `[0-9]{4}`

   Again matches **exactly 4 digits** after the hyphen.

8. `\b`

   Another **word boundary**.

   It ensures the serial number ends at the end of a word.

   **Example:**

   ```
   SN1234-5678      ✅
   SN1234-56789     ❌
   SN1234-5678ABC   ❌
   ```

9. The third argument `'c'`

   ```sql
   REGEXP_LIKE(description, pattern, 'c')
   ```

   enables **case-sensitive matching**, so only uppercase `SN` is accepted.

10. Return all matching products ordered by `product_id`.

---

### ⏱️ Complexity

**Time Complexity:** `O(n × m)`

- `n` = number of rows
- `m` = average length of the description

**Space Complexity:** `O(1)`
