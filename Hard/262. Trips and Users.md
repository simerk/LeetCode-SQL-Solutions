## 🚀 262. Trips and Users

### 💡 Idea

The goal is to calculate the **daily cancellation rate** for trips between **2013-10-01** and **2013-10-03**.

Only trips where **both the client and the driver are not banned** should be considered. We count the number of cancelled trips, divide it by the total number of valid trips for each day, and round the result to **2 decimal places**.

---

### 💻 Solution

```sql
SELECT
    t.request_at AS Day,
    ROUND( SUM(t.status <> 'completed') / COUNT(*), 2 ) AS `Cancellation Rate`
FROM Trips t
JOIN Users c ON t.client_id = c.users_id
JOIN Users d ON t.driver_id = d.users_id
WHERE c.banned = 'No' AND d.banned = 'No'
AND t.request_at BETWEEN '2013-10-01' AND '2013-10-03'
GROUP BY t.request_at;
```

---

### 🛠️ How It Works

1. Join the `Trips` table with the `Users` table twice:
   - Once to retrieve the **client** information.
   - Once to retrieve the **driver** information.
2. Filter out trips where either the client or the driver is banned.
3. Restrict the data to trips between **2013-10-01** and **2013-10-03**.
4. Group the trips by `request_at` to calculate the cancellation rate for each day.
5. `SUM(t.status <> 'completed')` counts the cancelled trips because:
   - `TRUE` → `1`
   - `FALSE` → `0`
6. Divide the number of cancelled trips by the total number of valid trips using `COUNT(*)`.
7. Round the result to **2 decimal places** using `ROUND()`.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(single scan with joins and grouping)*

**Space Complexity:** `O(1)` *(excluding the output result set)*
