## 🚀 3482. Analyze Organization Hierarchy

### 💡 Idea

The goal is to analyze the organization's hierarchy by finding:

- The **hierarchy level** of each employee.
- The **team size** (direct and indirect reports).
- The **salary budget** controlled by each employee.

We use two **Recursive CTEs**:

- The first CTE determines the hierarchy level of every employee.
- The second CTE finds all employees reporting (directly or indirectly) to each manager.

---

## 💻 Solution

```sql
WITH RECURSIVE hierarchy AS (
    SELECT employee_id, employee_name, manager_id, salary,
    1 AS level
    FROM Employees
    WHERE manager_id IS NULL
    UNION ALL
    SELECT e.employee_id, e.employee_name, e.manager_id, e.salary,
    h.level + 1 AS level
    FROM Employees e
    JOIN hierarchy h ON e.manager_id = h.employee_id
),
reports AS (
    SELECT employee_id AS manager_id, employee_id AS report_id, salary
    FROM Employees
    UNION ALL
    SELECT r.manager_id, e.employee_id AS report_id, e.salary
    FROM reports r
    JOIN Employees e ON e.manager_id = r.report_id
)
SELECT h.employee_id, h.employee_name, h.level,
(COUNT(r.report_id) - 1) AS team_size,        
SUM(r.salary) AS budget
FROM hierarchy h
JOIN reports r ON r.manager_id = h.employee_id
GROUP BY h.employee_id, h.employee_name, h.level
ORDER BY h.level ASC, budget DESC, h.employee_name ASC;
```

---

### 🛠️ How It Works

#### **Step 1: Build the hierarchy**

The `hierarchy` recursive CTE starts with the **CEO** (`manager_id IS NULL`) at **Level 1**.

```sql
1 AS level
```

For every recursive iteration,

```sql
h.level + 1
```

assigns the next hierarchy level to employees reporting to the current manager.

---

#### **Step 2: Find every reporting relationship**

The `reports` recursive CTE builds every **manager → employee** relationship.

First,

```sql
SELECT
employee_id AS manager_id,
employee_id AS report_id
```

treats every employee as their own report.

This makes it easy to include the manager's own salary when calculating the budget.

Then,

```sql
JOIN Employees e
ON e.manager_id = r.report_id
```

recursively finds:

- Direct reports
- Indirect reports

for every manager.

---

#### **Step 3: Calculate the team size**

```sql
COUNT(r.report_id) - 1
```

`COUNT()` includes the manager because of Step 2.

Subtracting **1** removes the manager, leaving only their team members.

---

#### **Step 4: Calculate the salary budget**

```sql
SUM(r.salary)
```

Since every manager is included in their own reporting chain, this automatically calculates:

- Manager's salary
- All direct reports' salaries
- All indirect reports' salaries

---

#### **Step 5: Return the result**

Finally,

```sql
ORDER BY
level,
budget DESC,
employee_name
```

sorts the output by:

1. Hierarchy level (ascending)
2. Salary budget (descending)
3. Employee name (ascending)

---

### ⏱️ Complexity

**Time Complexity:** `O(n²)`

**Space Complexity:** `O(n²)`
