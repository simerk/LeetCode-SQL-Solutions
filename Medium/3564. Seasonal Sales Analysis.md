## 🚀 3564. Seasonal Sales Analysis

### 💡 Idea

The goal is to find the **most popular product category** for each season.

The popularity of a category is determined by:

1. Highest **total quantity sold**.
2. If there's a tie, choose the category with the **highest total revenue**.
3. If there's still a tie, choose the **lexicographically smaller category**.

We first determine the season for each sale, calculate the total quantity and revenue for every **(season, category)** pair, and then use a **window function** to rank the categories within each season. Finally, we select the top-ranked category for each season. :contentReference[oaicite:0]{index=0}

---

## 💻 Solution

```sql
WITH SeasonalSales AS (
    SELECT
        CASE
            WHEN MONTH(sale_date) IN (12,1,2) THEN 'Winter'
            WHEN MONTH(sale_date) IN (3,4,5) THEN 'Spring'
            WHEN MONTH(sale_date) IN (6,7,8) THEN 'Summer'
            ELSE 'Fall'
        END AS season,
        p.category,
        SUM(s.quantity) AS total_quantity,
        SUM(s.quantity * s.price) AS total_revenue
    FROM Sales s
    JOIN Products p ON s.product_id = p.product_id
    GROUP BY season, p.category
),
RankedCategories AS (
    SELECT *,
        ROW_NUMBER() OVER (PARTITION BY season
        ORDER BY total_quantity DESC, total_revenue DESC, category ASC) AS rn
    FROM SeasonalSales
)
SELECT season, category, total_quantity, total_revenue
FROM RankedCategories
WHERE rn = 1
ORDER BY season;
```

---

### 🛠️ How It Works

1. Join the **Sales** and **Products** tables to get the category of each sale.
2. Use a `CASE` statement with `MONTH()` to assign every sale to its corresponding season.
3. Group by **season** and **category** to calculate:
   - Total quantity sold.
   - Total revenue (`quantity × price`).
4. Use `ROW_NUMBER()` to rank categories within each season by:
   - Highest quantity.
   - Highest revenue.
   - Lexicographically smallest category.
5. Select the category with `rn = 1` for each season and sort the final result by season.

---

### ⏱️ Complexity

**Time Complexity:** `O(n log n)`

- `O(n)` to join and aggregate the sales.
- Window ranking performs sorting within each season.

**Space Complexity:** `O(n)`

- Used to store the aggregated seasonal sales before ranking.
